Mobile Application Programming E-Portfolio

SDG 12: Responsible Consumption and Production

This e-Portfolio showcases the laboratory activities, projects, technical skills, and learning experiences gained throughout the Mobile Application Programming course. The main application developed for this course is EcoTracker, a sustainability-focused mobile application that encourages environmentally responsible habits through activity tracking, environmental awareness, and community engagement.

Lab Activities

This section presents the laboratory activities completed throughout the course. Each lab helped build my understanding of Android development step by step, from basic interface design to more advanced mobile application features.

Lab 1: Introduction to Android Development

Lab 1 introduced the basic development environment and the foundation of creating a mobile application using Android Studio. This activity helped me become familiar with project structure, interface components, and the basic process of running an Android application.

What I learned: I learned how Android projects are structured, how to work with the development environment, and how simple interface elements are arranged in a mobile application.

Lab 2: User Interface Design

Lab 2 focused on designing and improving the user interface of a mobile application. It involved arranging layout components, managing visual structure, and ensuring that the application interface is clear and easy for users to understand.

What I learned: I learned the importance of spacing, alignment, readable text, and consistent interface design in creating a better user experience.

Lab 3: Interaction and Application Flow

Lab 3 introduced interactive application features and user input handling. This lab helped me understand how users interact with application components and how the application should respond to user actions.

What I learned: I learned how interaction improves usability and how user input can be processed to make the application more functional.

Lab 4: Navigation Implementation

Lab 4 focused on navigation between application screens. This activity was important because navigation allows users to move smoothly between different sections of an application.

What I learned: I learned how screen navigation supports application flow and how multiple screens can be connected to create a complete mobile application experience.

Lab 5: Data Persistence Concept

Lab 5 introduced the concept of storing and managing application data. This lab helped me understand why data persistence is important in mobile applications, especially when user information needs to be saved and retrieved.

What I learned: I learned the role of data storage in mobile applications and how saved data can improve continuity and usefulness for users.

Projects

The following projects demonstrate the knowledge and skills developed throughout the course, including user interface design, navigation, cloud integration, API communication, and sensor implementation.

Project 1: EcoTracker

Problem Statement: Many individuals wish to adopt environmentally responsible habits but often struggle to monitor and maintain sustainable actions consistently in their daily lives.

EcoTracker was developed to support Sustainable Development Goal 12 (Responsible Consumption and Production) by encouraging users to record environmentally friendly activities and monitor their sustainability progress.

Lessons Learned: Through this project, I learned how to design a complete mobile application using Jetpack Compose, implement navigation between screens, organise application architecture, and create a user-friendly interface that supports sustainability awareness.
🐙 View on GitHub

Project 2: EcoTracker 2.0

Problem Statement: Users often require more engaging features and better access to environmental information to maintain long-term sustainable behaviour. Traditional activity tracking applications may lack real-time information, community interaction, and multimedia support.

Project 2 enhanced EcoTracker by integrating Firebase Firestore, REST API communication, camera functionality, image gallery support, environmental impact tracking, and community sharing features to create a more interactive sustainability platform.

Lessons Learned: This project strengthened my understanding of cloud databases, REST API integration, camera sensor implementation, image handling, and real-time data management. It also improved my ability to combine multiple Android development technologies into a single application.

Reflection

This section reflects on my learning experience throughout the Mobile Application Programming course.

Throughout this course, I developed my understanding of mobile application development using Android Studio, Kotlin, and Jetpack Compose. The laboratory activities helped me progressively strengthen my technical foundation, beginning with interface design and continuing towards navigation, data management, cloud integration, API communication, and sensor implementation.


The development of EcoTracker allowed me to apply theoretical knowledge in a practical environment by designing a complete mobile application that addresses a real-world sustainability issue. Through Project 1 and Project 2, I gained experience in planning application features, managing user interactions, integrating external technologies, and improving the overall user experience.


One of the most valuable lessons gained from this course was understanding how multiple technologies can work together within a single mobile application. Features such as Firebase Firestore, Retrofit API integration, image handling, camera functionality, and user activity tracking demonstrated how modern mobile applications provide dynamic and interactive experiences.


Overall, this course enhanced my technical confidence, problem-solving abilities, and understanding of Android application development. It also strengthened my appreciation for how technology can be used to encourage positive behavioural change and support sustainability initiatives.
